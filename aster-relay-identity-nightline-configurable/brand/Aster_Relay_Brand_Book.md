# Aster Relay — Brand Book

## Position

**Aster Relay** is an identity system for a mixed-gender tech collective that treats
education, privacy, and hands-on self-protection as shared civic practice. The tone is calm,
exact, mutual, and competent.

This revision starts from the uploaded **Midnight Galaxy** palette — deep purple, cosmic blue,
lavender, silver — and moves it into a sharper sibling system with indigo depth, expressive
violet structure, pixel rhythm, and a small neon signal. The result remains readable and
unobtrusive while gaining a little lived-in character.

## Name and line

- **Name:** Aster Relay
- **Line:** Learn quietly. Build carefully. Protect each other.
- **Descriptor:** practical privacy and mutual tech literacy
- **Style layer:** Nightline

## The Nightline edge

The system now carries three restrained accents:

1. **Simple 90s graffiti whisper**  
   Thin slanted underlines, spray-soft residue, and handstyle-like tags are allowed as secondary
   moves. They never replace the main typographic system.

2. **Pixel affection**  
   Step-shaped stars, dither fields, and visible grids create texture and memory without turning
   the brand into a retro toy.

3. **Subtle neon punch**  
   Cyan and magenta appear as pulse colors, not floods. One small glow is usually enough.

- **Recommended comment wrap:** 88 chars

## Writing posture

The visual edge does not license noisy copy. The group should still sound calm, direct, and
adult. Use exact titles and plain verbs. Keep metaphor density low. Let posters carry attitude;
let documentation carry trust.

## Icon direction

Icons are square-friendly, pixel-light, and structurally simple. Favor step forms over soft blobs.
Use one accent color plus one neutral support. Avoid over-detail.

## Documents and code

- Letters: formal layout, one signal accent only
- Docs CSS: indigo surfaces with restrained glow cues
- Markdown: narrow prose and clear heading ladder
- LaTeX: readable headings, dark-theme examples, configurable brand macros
- Source comments: intent-first, no swagger

## Configurable brand name

The pack now includes a configurable layer:

- `config/brand.json`
- `scripts/apply_brand.py`
- placeholder-based templates in `templates/configurable/`

This keeps the symbol system stable while allowing the collective name, slug, line, domain, and
contact details to be swapped quickly.
