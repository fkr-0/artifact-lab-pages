# Every item/char?

You do **not** have to design every character with every item. That is the trap.

I added a project note here:

```text
/home/user/work/code/artifacts/ethic-brawl/docs/design/equipment-rendering-shortcuts.md
```

Use this shortcut model:

```text
final frame =
  character body frame
  + hand anchor/socket
  + item overlay sprite
  + optional VFX/projectile
```

So instead of:

```text
16 characters × 9 items × all animation frames
```

you author:

```text
16 character base sheets
+ 9 item overlay sheets
+ small per-character/per-frame anchor metadata
```

Best structure for Ethic Brawl:

```text
assets/sprites/items/
  pipe.png
  uzi.png
  bat.png
  katana.png
  molotov.png
  grenade.png
  boulder.png
  rocket_launcher.png
  minidrone.png
```

Then define shared pose families:

```text
carry_light      pipe, bat, katana, grenade, molotov
carry_heavy      boulder, rocket launcher
swing_light      pipe, bat, katana
throw_light      grenade, molotov
shoot_1h         uzi
shoot_2h         rocket launcher
deploy           minidrone
pickup/drop      all items
```

For rendering:

```text
shadow
back arm / back clothing
back item part
body sprite
front item
front arm / hand cover
muzzle flash / slash arc / projectile
```

The current project is already close conceptually: `src/game/items/item-system.ts` separates item effects from characters, and `src/content/items/item-data.ts` already has weapon-family concepts like sword, bat, and mace. The next missing layer is an `ItemVisualDefinition` that maps an item to a sprite overlay, anchor, pose family, pivot, scale, and grip angle.

Use custom full redraws only for exceptions:

```text
rocket launcher bracing
boulder overhead carry
signature katana flourish
boss-only item stance
```

Everything else should be socketed overlays.

# ETHIC BRAWL — item/socket sprite authoring table

## 1. What every character needs

| Asset / metadata               | Required per character | Sprite-LLM prompt form                                                                                                                                                                                               |
| ------------------------------ | ---------------------: | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Base body sheet                |                    Yes | "Create a production-ready pixel-art character sprite sheet on true alpha. Character holds no item. Hands must be readable and separated enough to support item overlays. Keep all frames aligned to an exact grid." |
| Hand/socket map                |                    Yes | "For every frame, mark the front-hand grip point, back-hand grip point, palm direction, and whether the hand is in front of or behind the item."                                                                     |
| Pose family support            |                    Yes | "Include neutral, carry_light, carry_heavy, swing_light, thrust, throw_light, shoot_1h, shoot_2h, deploy, pickup, drop, stagger-with-item, and disarm-compatible poses."                                             |
| Front/back arm layering        |                    Yes | "Render body so item overlays can be layered between back arm and front arm. Avoid arms crossing the torso in ways that hide the grip unless intentional."                                                           |
| Contact shadows                |    Optional but useful | "Keep character contact shadow separate or minimal, so item shadows and thrown objects can be added consistently."                                                                                                   |
| Character scale contract       |                    Yes | "Keep character feet, head height, hand size, and camera angle consistent across all equipment-compatible frames."                                                                                                   |
| Item occlusion masks           |   Strongly recommended | "Create small hand-cover masks for front hand, back hand, and sleeve cuffs so item handles appear naturally gripped."                                                                                                |
| Hurt / knockback compatibility |                    Yes | "Include hurt and knockback poses where item can either remain attached to hand socket or detach into a dropped item sprite."                                                                                        |
| Facing direction rule          |                    Yes | "Design frames facing right by default. Left-facing sprites are generated by horizontal flip unless asymmetric details require manual override."                                                                     |
| Anchor metadata file           |                    Yes | "For each frame, output metadata containing socket coordinates, hand layer order, grip angle, and pose family."                                                                                                      |

## 2. Character prompt template

Use this whenever generating a new equipment-compatible character sheet.

```text
Create a production-ready pixel-art sprite sheet for ETHIC BRAWL.

FORMAT:
- True alpha transparent background.
- Exact grid.
- No gutters, no margins, no decorative borders.
- Character fully inside each cell.
- Consistent scale, camera angle, lighting, and foot baseline.
- No item held in the base body sheet unless the frame is explicitly an item-test frame.

EQUIPMENT COMPATIBILITY:
- Hands must be readable and usable as sockets.
- Every relevant frame must support item overlays.
- Leave enough visual space around hands for pipe, bat, katana, uzi, grenade, molotov, boulder, rocket launcher, or minidrone controller.
- Include clear front-hand and back-hand positions.
- Avoid hiding both hands inside clothing or behind the body.
- Arms should support front/back layering:
  1. back arm
  2. item
  3. front arm / hand cover

POSE FAMILIES:
- idle_unarmed
- carry_light
- carry_heavy
- swing_light
- thrust
- throw_light
- shoot_1h
- shoot_2h
- deploy
- pickup
- drop
- hurt_with_item
- disarmed

STYLE:
- Crisp retro pixel art.
- Strong silhouette.
- Small-size readability.
- No text, labels, UI, watermark, or background.
```

| Pivot point           |                    Yes | "Mark the pivot used for swings, rotations, recoil, and throw arcs."                                             |
| Layer rule            |                    Yes | "Define whether the item appears behind body, between arms, or in front of body for each pose family."           |
| Scale rule            |                    Yes | "Keep item scale consistent with existing character sprite proportions."                                         |
| Angle variants        |                    Yes | "Provide item variants at common angles: down-rest, forward, upward, back-swing, overhead, thrown, and impact."  |

| Projectile sprite     | For thrown/fired items | "Create projectile-flight frames separately from the held item."                                                 |
| Dropped pickup sprite |                    Yes | "Create a small ground/pickup version of the item, readable at game scale."                                      |
| Two-hand support      | For heavy/long weapons | "Provide front-hand and back-hand grip points."                                                                  |

## 7. Practical production rule

| Decision                 | Use shortcut overlay? |                          Custom redraw? |
| ------------------------ | --------------------: | --------------------------------------: |
| One-handed small item    |                   Yes |                                  Rarely |
| Baton, pipe, bat, katana |                   Yes |              Only for signature attacks |
| Grenade, molotov         |                   Yes |   Only throw release hand may need mask |
| Uzi                      |                   Yes |                   Maybe for recoil pose |
| Boulder                  |               Partial |     Often needs custom heavy-carry pose |
| Rocket launcher          |               Partial |       Needs custom shoulder-braced pose |
| Minidrone                |                   Yes | Deploy pose only needs hand/palm socket |
| Boss finisher with item  |                 Maybe |                             Usually yes |

Core rule:

```text
Draw characters as clean bodies with reliable hand sockets.
Draw items once as reusable overlays.
Only redraw the whole character when the item changes body mechanics.
```
